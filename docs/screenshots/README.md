## Screenshot Guidelines

### Image Format
- **Format:** PNG (preferred) or JPG
- **Resolution:** 1920x1080 or similar
- **Size:** Optimize images (under 500KB each)

### Best Practices
1. **Clean Browser:** Use incognito/private mode to avoid extensions
2. **Full Width:** Capture the full page content
3. **Actual Data:** Use realistic test data, not placeholder text
4. **Consistent Theme:** Keep the same browser zoom level
5. **No Personal Info:** Blur or redact any sensitive information

### Browser Settings
- **Zoom:** 100%
- **Window Size:** 1920x1080 or larger
- **Theme:** Use the application's default theme

### Tools for Screenshots
- **Windows:** Win + Shift + S (Snipping Tool)
- **Mac:** Cmd + Shift + 4
- **Chrome:** DevTools > More tools > Screenshot
- **Extensions:** Full Page Screen Capture, Awesome Screenshot

## After Capturing

1. Save all screenshots in this folder (`docs/screenshots/`)
2. Verify all filenames match exactly (case-sensitive)
3. The documentation will automatically reference these images
4. Test the documentation by viewing it in a Markdown viewer
